# iRacingExtraProperties Dependency Audit

Validated against commit: HEAD  
Last updated: 2026-05-29  
Branch: work

## Scope and edit policy

This audit searched for remaining references to optional `RSC.iRacingExtraProperties.dll` / `IRacingExtraProperties` / `iRacingExtraProperties` / `ExtraProperties` dependency points across code, documentation, and packaged dashboards.

Dashboard files were audited but intentionally not modified because `.simhubdash` / dashboard JSON payloads can diff as large single-line files and must be manually fixed in SimHub Dash Studio.

## Analysis summary

- Owning subsystem: dash integration contract, with affected historical/runtime surfaces in Pit Commands, Opponents/H2H presentation, CarSA player metadata, MSGV1/messaging warnings, and Pit Entry Assist fallback-removal warnings.
- Upstream dependencies: native SimHub `DataCorePlugin.*` telemetry/session data, existing `LalaLaunch.*`/`Opp.*`/`H2H*`/`Car.Player.*` exports, and plugin-owned pit command actions.
- Downstream consumers: packaged dashboards under `Dash Files/*.simhubdash`, user-facing docs, support templates, and runtime warning docs.
- Relevant recent changes: 2026-04-10 and later changelog entries already removed active runtime `IRacingExtraProperties` reads and redirected pit/dashboard guidance to plugin-owned actions.
- Invariants: no dashboard JSON/layout edits; no Fuel, Pit, H2H, CarSA, RaceFinish, Strategy, or telemetry math changes; no new optional plugin dependencies.
- Behavior change: none in this task. This audit adds documentation only and does not alter runtime code or dashboards.
- Risk: manual dashboard replacements can change display semantics when old ExtraProperties semantics do not have exact one-to-one Lala exports, especially theoretical/optimal lap, licence colour/text colour, and old gap-change colour conventions.
