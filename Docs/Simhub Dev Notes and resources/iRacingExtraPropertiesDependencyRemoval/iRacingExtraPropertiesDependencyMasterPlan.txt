IRACING EXTRA PROPERTIES DEPENDENCY MASTER HANDOVER
Pruned-design version
Backups excluded from dash scan

Purpose

This document defines the current dependency on iRacing Extra Properties across plugin code and active dashes, but with a v1-release lens:
- first identify removable debug, legacy, and redundant fallback usage
- then isolate the final critical dependency surface that truly needs internal replacement

This document is therefore split into:
1. removable dependencies
2. dependencies to retain for now
3. final must-replace surface after cleanup

==================================================
1. HIGH-LEVEL CONCLUSION
==================================================

The current dependency picture is noisier than the true engineering need.

A meaningful portion of Extra Properties usage appears to be:
- debug-only
- legacy-only
- convenience fallback only
- metadata fallback only

Before designing an internal replacement layer, LalaLaunch should first remove dependency sites that no longer justify their maintenance cost.

The real long-term replacement target is smaller and centers on:
- stable player race identity
- same-class leaderboard row stream
- race-order-aware neighbor selection
- player lap-percent feed for Opponents pit-exit cadence
- selected traffic/message signals if still in v1 scope
- selected dash-facing race/traffic properties still used live

==================================================
4. POST-PRUNING TRUE MUST-REPLACE SURFACE
==================================================

Assuming the removable items above are actually cleaned out, the core replacement surface becomes:

A. Must replace
1. Stable player race identity
Needs:
- class color
- car number

2. Internal same-class leaderboard row stream
Needs:
- stable row identity
- overall position
- class position
- pit state
- connected state
- last lap
- best lap
- internal gap basis equivalent to RelativeGapToLeader

3. Race-order-aware same-class neighbor selection
Needs:
- player row lookup
- same-class ahead/behind selection by race order
- preservation of Opp.* and H2HRace.* contracts
- no fallback to simple proximity for race mode

4. Player lap-percent feed for Opponents cadence
Needs:
- stable player lap progress source for pit-exit timing/prediction cadence

B. Likely must replace if retained in v1
5. Fuel laps remaining source
- current dependency: iRacing_LapsRemainingFloat
Retain on must-replace list unless fuel design changes first

6. MsgV1 traffic/race-control signals
- nearest behind gap
- nearest behind distance
- nearest ahead gap
- slowdown timer
Retain on must-replace list only if MsgV1 remains a release-facing v1 feature

7. Dash-facing race target properties
Retain on must-replace list where active dashes still depend directly on Extra Properties semantics, especially:
- iRacing_DriverAhead_00_*
- iRacing_DriverBehind_00_*
- iRacing_DriverBehind_01_*
- iRacing_DriverBehind_02_*
- iRacing_DriverBehindInClass_00..02_CarNumber
- iRacing_NonRelativeDriverAheadInClass_00_*
- iRacing_NonRelativeDriverBehindInClass_00_*

==================================================
5. RECOMMENDED NEXT STEP
==================================================

Do not start internal replacement design yet.

First run a bounded cleanup task:
- remove debug-only dependencies
- remove confirmed dead mappings
- remove redundant fallbacks that are no longer needed for v1

Then rerun the dependency inventory.

Only after that should the final replacement design be frozen, because the post-cleanup dependency surface will be smaller, clearer, and much safer to architect around.

==================================================
6. BOTTOM LINE
==================================================

The right question is no longer:
“What Extra Properties do we use?”

The better v1 question is:
“What Extra Properties still justify existing after we remove debug and redundant fallback baggage?”

Once that pruning is done, the true core problem is likely just:
- race identity
- same-class leaderboard ownership
- race-order-aware neighbor selection
- lap-progress cadence
- fuel/message/dash extras only where still deliberately retained
