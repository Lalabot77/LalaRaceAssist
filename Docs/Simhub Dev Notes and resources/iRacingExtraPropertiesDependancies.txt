4. HIGH-LEVEL REPLACEMENT GROUPS
========================================

Likely easiest direct replacements
- player metadata
- lap progress
- class position
- pit distance / pit state related items
- car number / class identity where already exposed by LalaLaunch or SimHub raw data

Likely medium effort
- relative nearby behind slots used by rejoin / alerts / traffic widgets
- ahead/behind licensing / rating / class display enrichments

Likely hardest / most important race-intelligence replacements
- NonRelativeDriverAheadInClass_00_*
- NonRelativeDriverBehindInClass_00_*
- any same-class race-order target logic that depends on Extra Properties semantics
- gap-to-player and gap-change semantics that users already trust visually

Bottom line
The active dash layer is still directly dependent on iRacing Extra Properties in multiple core widgets, especially:
- rejoin widgets
- alerts widgets
- traffic displays
- OnTrackOpp
- Racing AHD BHD
- main message dash
- pit popup / pit hit vertical elements for some pit-related values

This list is the active dash-side replacement surface.
It should later be merged with the plugin/code dependency inventory to create the final master Extra Properties replacement register.
